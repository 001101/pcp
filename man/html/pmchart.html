Notes for pmchart users

Users of the original SGI pmchart utility will be immediately at home with kmchart. This page discusses two areas where differences are more likely to be noticed by people familiar with pmchart.

Configuration file syntax

kmchart provides the ability to parse and interpret all versions of the pmchart configuration file language (i.e. versions 1, 1.1, 1.2 and 2.0. In addition, kmchart provides it's own configuration file syntax, which both addresses some of the quirkier aspects of the pmchart language and provides support for features specific to kmchart (Legend Labels and Tabs, for example).

Refer to the discussion of kmchart Views for a detailed description of the kmchart configuration language.

Finally, while kmchart will read all pmchart configuration files, it does not implement support for writing them - only the native kmchart is written during a View save operation.

User interface changes

While every effort has been made to keep all of the excellent features of the pmchart user interface, kmchart provides support for several new user interface advances above and beyond everything pmchart provided.

A complete list of these new features can be found on the What's new in kmchart page. It includes some functionality which was sought after in pmchart but never implemented there (for example, the metric search function in the revised metric selection dialog).
